
## Nomenclature

The project follows a coherent geological and lacustrine naming convention. Components are named for the sedimentary processes of deep glacial lakes — systems defined by layered deposition, temporal stratification, and the preservation of what came before.

| Component | Name | Description |
|---|---|---|
| FUSE Prototype | **Alluvium** | Named for sediment deposited by flowing water; material that accumulates and layers over time. The Phase 1 proof of concept for the storage and versioning layers. |
| Kernel Filesystem | **Varvefs** | Named for varve sediment layers — thin, annually deposited strata in glacial lakes. Evokes the temporal and versioning characteristics of the system. |
| Operating System | **DeepLake OS** | The full operating system vision built on Varvefs. |
| Garbage Collection Daemon | **diagenesisd** | Named for diagenesis — the geological process by which loose sediment transforms into consolidated rock over time. Gradual, deliberate compaction. |

Future components should draw from the same vocabulary. Candidate terms identified for remaining unnamed components (metadata daemon, enrichment daemon, sync daemon, CLI tooling):

- **Zone separation:** thermocline, epilimnion, hypolimnion
- **Data lifecycle:** turbidite, laminae, detritus, diagenesis
- **Data provenance:** allochthonous (ingested), autochthonous (natively created)
- **Flow and deposition:** fluvial, deltaic
- **Glacial (consistent with varve):** moraine, ablation
